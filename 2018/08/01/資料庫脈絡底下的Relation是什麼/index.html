資料庫脈絡底下的Relation是什麼

Posted on 2018-08-01 | In 資料庫

Relation

  • 理論背景: Relational Database Theory
  • 理論提出者: E. F. Codd 於1970年左右

Relational Database 是建立在 relational model 這類 data model 之上的一種資料庫架構。其他類型的data model還有 network model 和 hiearchal model。

所謂data model是指為世界上的人事物擷取特定屬性、表達彼此關係、建立框架模型，以達成特定目的。
以城市地圖為例，城市地圖的目的是要讓開車的人或觀光客知道「路怎麼走」、「要走多遠」、「有哪些商店地標」…等等，因此地圖不需要完美的複製整座城市所有的資訊，像是路樹、路燈…等資訊不用畫在地圖上。相反的，比例尺、停車場、單行道、高速公路、方位…等資訊就非常關鍵。
所以城市地圖是城市的其中一種data model。

以表格為例

名字 年齡 性別 職業
小王 20 男 學生
小美 30 女 廚師

上面這個表格(table)的構成成份可看成：

  • 2個row：(小王, 20, 男, 學生), (小美, 30, 女, 廚師)
  • 4個column：(名字, 年齡, 性別, 職業)

把 table, row, column 這幾個SQL的術語轉換成Relational datbase的術語：

SQL用語 Relational DB 用語 描述
row tuple 或 record 反映著一件事物的資料集合，在上述的例子中 (小王, 20, 男, 學生) 這個資料集合對應著某個人
column attribute 或 field 資料的標籤、屬性
table relation 或 base relvar 由屬於相同attribute的tuples 所構成的集合
