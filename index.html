Haskell中的polymorphic-function

Posted on 2018-08-01 | In Haskell

所謂polymorphic function(多態函數)指的是可作用於多種類型的輸入的函數。
以length函數為例：

length :: [a] -> Int

length函數是用來計算list長度，不管輸入是整數的list、字元的list、tuple的list…，因為這個函數在乎的是包含幾個元素，而非哪種元素，因此length被定義為一個多態的函數。

length函數簽名中的 type variable a 即是用以指涉它的函數輸入接受各種元素類型的list。

SQL

Posted on 2018-08-01 | In 資料庫

全名

SQL 是 Structured Query Language 的縮寫。

用途

SQL是一種用來處理紀錄在 RDBMS (Relational Database Management System 關聯式資料庫管理系統，例如：MySQL, Oracle, Microsoft Access, PostreSQL) 當中資料的語言。

依據用途可以分成以下四種子語言：

  • DDL (Data Definition Language)
    用來定義、修正、刪除資料表的架構
    • CREATE
    • ALTER
    • DROP
    • TRUNCATE
  • DML (Data Manipulation Language)
    對DB中的資料進行新增、刪除、更新的動作
    • INSERT
    • UPDATE
    • DELETE
    • MERGE
  • DQL (Data Query Language)
    用來查詢DB中的符合特定條件的資料
    • SELECT
    • FROM
    • WHERE
  • DCL (Data Control Language)
    授與特定權限、控制DB操縱事務的發生等等
    • GRANT
    • ROLLBACK
    • COMMIT

資料庫脈絡底下的Relation是什麼

Posted on 2018-08-01 | In 資料庫

Relation

  • 理論背景: Relational Database Theory
  • 理論提出者: E. F. Codd 於1970年左右

Relational Database 是建立在 relational model 這類 data model 之上的一種資料庫架構。其他類型的data model還有 network model 和 hiearchal model。

所謂data model是指為世界上的人事物擷取特定屬性、表達彼此關係、建立框架模型，以達成特定目的。
以城市地圖為例，城市地圖的目的是要讓開車的人或觀光客知道「路怎麼走」、「要走多遠」、「有哪些商店地標」…等等，因此地圖不需要完美的複製整座城市所有的資訊，像是路樹、路燈…等資訊不用畫在地圖上。相反的，比例尺、停車場、單行道、高速公路、方位…等資訊就非常關鍵。
所以城市地圖是城市的其中一種data model。

以表格為例

名字 年齡 性別 職業
小王 20 男 學生
小美 30 女 廚師

上面這個表格(table)的構成成份可看成：

  • 2個row：(小王, 20, 男, 學生), (小美, 30, 女, 廚師)
  • 4個column：(名字, 年齡, 性別, 職業)

把 table, row, column 這幾個SQL的術語轉換成Relational datbase的術語：

SQL用語 Relational DB 用語 描述
row tuple 或 record 反映著一件事物的資料集合，在上述的例子中 (小王, 20, 男, 學生) 這個資料集合對應著某個人
column attribute 或 field 資料的標籤、屬性
table relation 或 base relvar 由屬於相同attribute的tuples 所構成的集合

在二維平面上，每個向量 $\vec{v}$ 都是以基向量來定義：

$\hat{i}=\pmatrix{1\\0}$ 為在x軸方向的基向量
$\hat{j}=\pmatrix{0\\1}$ 為在y軸方向的基向量

說 $\vec{v} = \pmatrix{3\\2}$ 即是說「對$\hat{i}$放大3倍的向量加上對$\hat{j}$放大兩倍的向量」：

$\vec{v} = 3\hat{i} + 2\hat{j} = 3\pmatrix{1\\0} + 2\pmatrix{0\\1} = \pmatrix{3\\2}$
